# Настройка Google Gemini API

## Получение API ключа

1. Откройте [Google AI Studio](https://aistudio.google.com/)
2. Войдите в свой Google аккаунт
3. Нажмите на кнопку **"Get API Key"** (Получить API ключ) в левом меню
4. Выберите проект или создайте новый
5. Скопируйте созданный API ключ

**Важно:** 
- Gemini API ключи начинаются с `AIza` и имеют длину около 39 символов
- Если ваш ключ начинается с `AQVN` или другой последовательности - это ключ от другого сервиса (например, Yandex)
- Убедитесь, что вы получили ключ именно из Google AI Studio

## Настройка проекта

1. Создайте файл `.env` в корне проекта (или скопируйте `.env.example`)
2. Добавьте ваш API ключ:
   ```env
   GEMINI_API_KEY=ваш_api_ключ_здесь
   GEMINI_MODEL=gemini-pro
   ```

## Доступные модели

- **`gemini-pro`** - стандартная модель (рекомендуется)
- **`gemini-1.5-pro`** - более новая версия с расширенным контекстом
- **`gemini-1.5-flash`** - быстрая модель для простых задач
- **`gemini-pro-vision`** - модель с поддержкой изображений

**Важно:** Используйте точные названия моделей. Неправильное название модели приведет к ошибке 400.

## Проверка работы

После настройки проверьте работу API:

```bash
# Запустите прокси-сервер
npm run proxy

# В другом терминале или браузере откройте:
http://localhost:3001/test
```

Если все настроено правильно, вы увидите ответ от Gemini API.

## Ограничения и лимиты

- Бесплатный тариф: до 60 запросов в минуту
- Платные тарифы: более высокие лимиты
- Подробнее: [Gemini API Pricing](https://ai.google.dev/pricing)

## Решение проблем

### Ошибка 401 (Unauthorized)
- Проверьте правильность API ключа в файле `.env`
- Убедитесь, что ключ активен в Google AI Studio

### Ошибка 403 (Forbidden)
- Проверьте права доступа к API в Google Cloud Console
- Убедитесь, что API включен для вашего проекта

### Ошибка 429 (Too Many Requests)
- Превышен лимит запросов
- Подождите немного и попробуйте снова
